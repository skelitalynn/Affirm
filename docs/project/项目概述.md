# Affirm — 显化导师 AI Agent

**更新日期**：2026-03-05
**项目状态**：已部署，持续迭代

---

## 项目定位

Affirm 是一个基于 Telegram Bot 的个人 AI 显化导师助手。用户通过 Telegram 进行长期对话，系统具备向量语义记忆、知识库注入、每日对话归档（Notion）以及 Web 管理后台等能力。

---

## 核心特性

| 特性 | 状态 | 说明 |
|------|------|------|
| Telegram Bot 对话 | ✅ 生产可用 | Polling 模式，用户级串行队列 |
| 多 AI Provider | ✅ 生产可用 | DeepSeek / Claude / OpenAI 可切换 |
| 消息持久化 | ✅ 生产可用 | PostgreSQL，含 pgvector 向量列 |
| 向量 Embedding | ✅ 生产可用 | 独立 Embedding Provider（`EMBEDDING_API_KEY`），与主 AI Provider 解耦 |
| 语义记忆检索 | ✅ 生产可用 | RAG 已接入对话链路，历史消息语义检索启用 |
| 知识库注入 | ✅ 生产可用 | Admin 界面录入，RAG 自动检索并注入 Prompt |
| Notion 归档 | ✅ 可用 | `/archive_now` 命令手动触发 |
| Web 管理后台 | ✅ 可用 | Express + EJS，JWT 认证 |

---

## 技术栈

| 层次 | 技术 |
|------|------|
| 运行时 | Node.js 18+（CommonJS）|
| Bot 框架 | node-telegram-bot-api（Polling 模式）|
| AI 客户端 | openai npm 包（统一兼容接口）|
| 数据库 | PostgreSQL 15+ + pgvector 0.8.x |
| 管理后台 | Express 4 + EJS + Helmet |
| 配置管理 | dotenv + 自研 ConfigManager |
| 错误处理 | 自研 ErrorHandler + 自定义错误类 |
| 并发控制 | 自研 MessageQueue（用户级串行）|
| 容器化 | Docker + docker-compose |
| 进程管理 | PM2 |

## AI Provider 配置

通过 `AI_PROVIDER` 环境变量选择提供商：

```bash
AI_PROVIDER=deepseek    # DeepSeek（默认）
AI_PROVIDER=claude      # Claude via AiGoCode 代理
AI_PROVIDER=openai      # OpenAI 官方
```

所有 Provider 统一使用 OpenAI 兼容接口，无需更换代码。

---

## 环境变量（核心）

```bash
# 数据库
DB_URL=postgresql://user:password@localhost:5432/affirm

# Telegram
TELEGRAM_BOT_TOKEN=your_bot_token

# AI Provider
AI_PROVIDER=deepseek
DEEPSEEK_API_KEY=your_deepseek_key

# 可选：Claude
CLAUDE_API_KEY=your_claude_key
CLAUDE_BASE_URL=https://api.aigocode.com/v1

# 可选：Notion 归档
NOTION_TOKEN=your_notion_token
NOTION_PARENT_PAGE_ID=your_page_id

# Embedding（RAG 语义检索，独立于主 AI Provider）
EMBEDDING_API_KEY=your_openai_api_key
EMBEDDING_MODEL=text-embedding-3-small
EMBEDDING_DIMENSIONS=768

# 安全
JWT_SECRET=your_jwt_secret
CORS_ORIGINS=http://localhost:3000
```

完整环境变量模板见 `.env.example`。

---

## 快速开始

```bash
# 1. 复制环境变量模板
cp .env.example .env

# 2. 填写必要配置（DB_URL、TELEGRAM_BOT_TOKEN、AI Key）

# 3. 启动服务（Docker）
docker-compose up -d

# 4. 或本地启动
npm install
npm start
```
